# CDEK News Feed

Это небольшой новостной портал на React. В приложении я сделала несколько вариантов новостного блока, которые получают данные из API, поддерживают пагинацию и корректно обрабатывают состояния загрузки, ошибки и пустого списка.

## Ссылки

- GitHub: [https://github.com/ulianabocheeva/cdek-news-feed](https://github.com/ulianabocheeva/cdek-news-feed)
- Демо: [https://cdek-news-feed.vercel.app/](https://cdek-news-feed.vercel.app/)

## Что реализовано

- блок `Новости компании`
- блок `Бизнес`
- пустое состояние для блока `Важные новости`
- загрузка новостей из API
- пагинация между страницами
- skeleton во время загрузки
- сообщение об ошибке с возможностью повторить запрос
- адаптивная версия для мобильных устройств
- lazy loading изображений
- кэширование уже загруженных страниц

## Дополнительно

Я добавила несколько вещей, которые делают интерфейс удобнее:

- предзагрузку следующей страницы в фоне
- запоминание текущей страницы блока через `sessionStorage`
- разные шаблоны карточек для разных типов новостного блока
- плавное появление контента

## Настройка API

По умолчанию проект использует тестовый API:

```env
VITE_API_BASE_URL=http://1e14c3489fcb.vps.myjino.ru:5000/api/v1
```

Если нужно работать с локально поднятым API, можно создать `.env` на основе `.env.example` и указать:

```env
VITE_API_BASE_URL=http://localhost:5000/api/v1
```

В режиме разработки запросы к API и изображениям проходят через proxy Vite.

## Почему я сделал именно так

- Логику загрузки данных я вынесла в хук `useNewsFeed`, чтобы не смешивать UI и работу с API.
- Кэш страниц реализовала через `Map`, чтобы не отправлять повторные запросы при возврате на уже открытую страницу.
- Для изображений отдельно обработала относительные пути, которые приходят из API.
- Универсальность блока сделала через параметры `variant`, `endpoint` и `subtitleMode`.
- Для разных сценариев отображения сделала отдельные шаблоны карточек, чтобы интерфейс был ближе к макету.
